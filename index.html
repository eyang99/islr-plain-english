How to use this

Three passes per chapter

  1. Read the "Big Idea" — one paragraph. If it doesn't click, the rest won't either. Re-read.
  2. Play with the visualizations. Move sliders. Watch what breaks. Intuition lives in your fingers.
  3. Flip the flashcards at the bottom. If you can't explain the back of the card to a stranger, you don't know it. Go back up.

The point isn't to read fast. It's to close the tab and still be able to draw the picture on a napkin.

If you only remember five things

  1. Test error, not training error. A model that nails the training set and chokes on new data learned the noise, not the signal.
  2. Bias-variance tradeoff. Every model error is either too simple to see the pattern or too wiggly and chasing noise. You pay for one with the other.
  3. Cross-validation. The honest way to guess how a model will do on data it hasn't seen yet.
  4. Regularization. When you have too many knobs, tell the model "prefer small knobs." Ridge and lasso are just different flavors of that preference.
  5. More tests → more false positives. Run 100 tests at p < 0.05 and you expect ~5 fake "discoveries." Correct for it or pretend you didn't see it.
