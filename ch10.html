Chapter 10

Deep Learning

Neural networks, demystified. They're stacked logistic regressions. They learn features automatically. They dominate where no hand-crafted features exist (images, text, audio). They don't always win tabular data.

Big idea

A neural network is a chain of simple transformations that together can approximate any function.

Each layer takes a vector, multiplies by a matrix, adds a bias, passes it through a non-linear "activation." Stack enough of those and you have a universal function approximator. Train it by gradient descent on a loss. That's the whole game.

1. The single-layer neural network

Start with inputs $X_1, \ldots, X_p$. Build $K$ intermediate units ("hidden units"):

$$ A_k = g\!\left(\beta_{k0} + \sum_{j=1}^p \beta_{kj} X_j\right) \quad k = 1, \ldots, K $$

where $g$ is a non-linear activation function (ReLU, sigmoid, tanh). Then combine the hidden units into the final output:

$$ f(X) = \gamma_0 + \sum_{k=1}^K \gamma_k A_k $$

Each hidden unit is a little committee member who listens to the inputs, forms a weighted opinion, and applies a non-linear squash. The final output is a weighted average of the committee's opinions. More hidden units = bigger committee = more expressive.

Why the non-linearity is essential

If $g(z) = z$ (linear activation), the whole network collapses to a single linear regression. The non-linearity is what gives the network its universal approximation power. Without it, you're just doing matrix algebra with extra steps.

2. Activation functions

  • Sigmoid $\sigma(z) = 1/(1+e^{-z})$ — smooth, (0,1). Rarely used in hidden layers now (gradients vanish for big |z|).
  • tanh $\tanh(z)$ — smooth, (−1,1). Same vanishing-gradient issue.
  • ReLU $\max(0, z)$ — zero for negatives, linear for positives. Current default for hidden layers. Cheap, avoids vanishing gradients.
  • Softmax — for the output layer of multi-class classification. Turns a vector into a probability distribution.

3. Multi-layer (deep) networks

Stack hidden layers: the outputs of one layer become the inputs of the next. With L layers, the $l$-th layer computes:

$$ A^{(l)} = g\!\left(W^{(l)} A^{(l-1)} + b^{(l)}\right) $$

Why go deep instead of wider? Empirically, multiple narrower layers learn hierarchical features: early layers learn simple patterns (edges), later layers combine them (textures → parts → objects). One very wide shallow layer can in principle represent the same functions, but much less efficiently in practice and with much worse generalization.

4. How training actually works

Loss function

Squared error for regression, cross-entropy for classification. Same losses as their shallow cousins, applied to the network's output.

Gradient descent

Adjust every weight in a direction that reduces the loss. Take a small step. Repeat. For small networks, the gradient is computable. For large ones, it's computed efficiently via backpropagation — chain rule applied layer by layer, going backwards from the output.

Stochastic gradient descent (SGD)

Instead of using the full dataset for every step (expensive, memory-hungry), use a random minibatch. Noisier gradients, but faster per step, and the noise often helps escape bad local minima.

Modern optimizers

Adam and its variants adapt the learning rate per parameter. Usually faster than plain SGD. SGD with momentum still wins for final generalization in some settings. Practical default: Adam, then refine.

5. Regularization for networks

Networks have millions of parameters. Without regularization they memorize. Tools:

  • Weight decay: L2 penalty on weights. Ridge regression at scale.
  • Dropout: during training, randomly zero out a fraction of hidden units each pass. Forces redundancy. Like bagging many sub-networks.
  • Early stopping: watch validation loss; stop when it plateaus. Acts like implicit regularization.
  • Data augmentation: synthetically enlarge the training set by rotating, flipping, cropping, etc. The cheapest and often most effective.

6. Convolutional neural networks (CNNs) — for images

A dense layer connects every input to every hidden unit. For a 224×224 image, that's ~50,000 inputs per layer, hundreds of millions of weights. Wasteful — most image patterns are local and translation-invariant.

CNNs bake in that prior:

  • Convolution layer: slide a small filter (say 3×3) across the image. Same filter everywhere (shared weights). Each filter detects one kind of local feature (edge, curve).
  • Pooling layer: downsample by taking max or average in each small region. Makes the representation smaller and adds a little translation invariance.
  • Stack these: early filters detect edges; deeper filters detect textures, parts, objects.

CNNs turned image classification from impossible to solved between 2012 and 2015.

7. Recurrent neural networks (RNNs) — for sequences

For time series or text, you want the network to remember what came before. RNNs feed each hidden state forward as an extra input to the next step:

$$ h_t = g(W_h h_{t-1} + W_x x_t + b) $$

Same $W_h, W_x$ at every step (weight sharing over time). The hidden state accumulates memory.

Problem: vanishing gradients

Long sequences cause gradients to shrink or explode. Workarounds: LSTM and GRU cells add gates that protect long-term memory. Then transformers (self-attention) replaced RNNs for most sequence tasks after 2017, but the book's coverage focuses on LSTM/GRU.

8. When do neural nets win?

SituationBest bet
Tabular data, n in thousands, p modestGradient boosted trees (Ch 8)
Images, raw pixelsCNN
Text, raw tokensTransformer (post-book; the direction deep learning went)
Small data with domain knowledgeHand-crafted features + regularized linear/GBM
You need interpretabilityLinear / GAM / small tree
Deep learning is a weapon. It's overkill for most tabular problems and unrivaled for perception tasks (images, audio, text). ISLR is refreshingly honest that GBM still competes with — and sometimes beats — neural nets on tabular data.

Flashcards

1
Write the equation for one hidden unit.
A = g(β₀ + Σ βⱼ Xⱼ). Weighted sum of inputs plus bias, passed through a non-linear activation g.
2
Why are non-linear activations essential?
Without them, stacking layers is algebraically equivalent to one linear transformation. The non-linearity is what gives networks their universal approximation power.
3
What is backpropagation?
Chain rule applied to compute the gradient of the loss with respect to every weight, efficiently, by moving backward layer by layer from the output.
4
What is SGD?
Stochastic gradient descent: update weights using the gradient computed on a small random batch instead of the full dataset. Faster and noisier, which helps generalization.
5
What does dropout do?
During training, randomly zero out a fraction of hidden units each forward pass. Prevents co-adaptation, acts like ensembling many sub-networks.
6
Why use convolution layers for images?
Image patterns are local and translation-invariant. Convolutions share weights across locations and detect local patterns — far fewer parameters than a dense layer, with a better prior.
7
What's the vanishing gradient problem in RNNs?
Gradients multiplied through many time steps shrink (or explode) exponentially, so the network can't learn long-range dependencies. LSTMs/GRUs add gates to preserve information.
8
When is gradient boosting competitive with deep learning?
Tabular data with modest n. On images/text/audio, deep learning wins by a huge margin because it can learn features from raw input.
9
Why might deeper networks generalize better than wider shallow ones?
Depth encourages hierarchical feature composition (edges → parts → objects), which is a useful inductive bias and requires exponentially fewer parameters than a shallow network representing the same function.
