Chapter 5

Resampling Methods

Honest ways to measure a model's test performance without a separate test set, and honest ways to quantify uncertainty without hard math.

Big idea

Pretend you have more data than you do by reusing your data cleverly.

Cross-validation pretends you have a test set by holding pieces out in turn. The bootstrap pretends you have many datasets by resampling with replacement. Both substitute computation for data — the trade the whole field rests on.

1. The problem

You want to estimate test error (how your model does on new data) but you only have one dataset. If you train and evaluate on the same data, you get an optimistically biased estimate (your model has seen everything). If you save a chunk as a test set, you're throwing away training data you can't afford to lose.

Cross-validation solves this with a rotating hold-out. The bootstrap handles a different problem: estimating the standard error of anything, not just a mean.

2. Validation set approach — the naive start

Split data randomly into two halves: train on one, test on the other. Simple. Problems:

  • High variance. A different random split gives a different estimate.
  • Less training data. Model is fit on half the data, so it's worse than what you'd actually deploy.

Good as a teaching stepping stone. Don't use it in practice unless your dataset is huge.

3. Leave-One-Out Cross-Validation (LOOCV)

Take out exactly one observation. Train on the remaining n−1. Predict the held-out one. Record the squared error. Repeat for every observation. Average.

$$ CV_{(n)} = \frac{1}{n} \sum_{i=1}^n (y_i - \hat{y}_i^{(-i)})^2 $$

Pros: nearly unbiased (each model uses almost all the data).

Cons: expensive (n model fits). High variance because the n fitted models are nearly identical — highly correlated — so averaging them reduces variance less than you'd hope.

For least-squares linear regression, LOOCV has a closed form using the hat matrix — you fit the model once. For most other models, you have to actually do n fits.

4. k-fold cross-validation — the one you'll actually use

Split the data randomly into k equal folds. For each fold: hold it out, train on the other k−1, predict, record. Average the k errors.

Typical choices: k = 5 or 10. The bias-variance trade-off of k itself:

  • Small k (e.g., 2 or 3): more bias (each fold is trained on less data), less variance.
  • Large k (e.g., n = LOOCV): less bias, more variance (highly correlated fold errors).
  • k = 5 or 10 hits a sweet spot empirically.

5. CV for classification

Same idea, but instead of MSE you average misclassification rates (or log-loss, or AUC, depending on what you optimize). Use stratified folds so class proportions match in each fold — critical for imbalanced data.

Gotcha: if you do feature selection, scaling, or any choice using the full dataset before CV, you've leaked test information into training. The correct pattern: every preprocessing step goes inside the CV loop.

6. The bootstrap

Different problem: you have an estimate (a mean, a coefficient, a portfolio weight) and you want its standard error — but you don't have a clean formula. The bootstrap is the computational hammer.

The recipe

  1. Your dataset has n observations.
  2. Draw a "bootstrap sample" by sampling n times with replacement from it. Some observations appear twice, others not at all.
  3. Compute your statistic on this bootstrap sample. Record it.
  4. Repeat 1000+ times.
  5. The standard deviation of your recorded statistics = bootstrap estimate of the SE.
Imagine you only got to do the study once, but God shrugged and said "fine, I'll let you resample from your own sample, but don't tell anyone." The bootstrap fakes having many datasets by shuffling the one you have — with replacement, so you actually get variation.

Why it's almost magic

Most parametric SE formulas require you to know the underlying distribution (Gaussian, etc.). The bootstrap doesn't. It works for medians, correlations, ratios, regression coefficients, nearly anything. As long as the plug-in principle holds (your data is representative), the bootstrap gives you sensible uncertainty.

The bootstrap is not a great replacement for cross-validation when estimating test error. In a bootstrap sample, ~37% of original observations are missing — but many observations appear multiple times in both "training" and "evaluation," which contaminates the error estimate. Use k-fold CV for test error; use bootstrap for standard errors.

7. One-sentence mental model

Cross-validation answers "how well will my model generalize?" Bootstrap answers "how uncertain is this estimate?" Both reuse the data to give you something that used to require more data.

Flashcards

1
Why is training error a bad estimate of test error?
The model has already seen those points. A flexible enough model can drive training error to zero while generalizing poorly. Training error is optimistically biased.
2
Describe LOOCV in two lines.
For each observation i: train on the other n−1, predict y_i, record error. Average across all n. Nearly unbiased but computationally expensive.
3
Why do practitioners prefer k=5 or k=10 over LOOCV?
Lower variance in the estimate (folds are less correlated), cheaper to compute, and in practice the small bias increase doesn't matter much.
4
What's the common CV mistake that leaks test data?
Doing feature selection, normalization, or hyperparameter search on the full dataset before CV. All preprocessing must happen inside each fold's training step.
5
Describe the bootstrap in three lines.
Sample n observations with replacement from your dataset. Compute your statistic on the resample. Repeat 1000+ times. The SD of those statistics is the bootstrap SE.
6
Why is the bootstrap so general?
It doesn't need a distributional assumption. Works for any statistic as long as the sample is representative — means, medians, correlations, regression coefs, anything.
7
Why shouldn't you use bootstrap for test-error estimation?
About 37% of original observations are missing from each bootstrap sample, but many overlap between "train" and "test." The overlap makes the error estimate biased low. Use k-fold CV instead.
8
CV vs bootstrap — one-line difference.
CV estimates how well a model generalizes. Bootstrap estimates the uncertainty in a specific computed statistic.

Quiz

Resampling is how you buy honesty without a bigger dataset.

Question 1 · compute

You do 10-fold cross-validation on 1000 observations. How many observations are in each validation fold?

Solution

$1000 / 10 = \mathbf{100}$ per fold. In each of 10 rounds, one fold (100 obs) is the validation set and the other 900 train the model. The CV error is the average across rounds.

Question 2 · compute (bootstrap)

In a bootstrap sample of size $n$ drawn with replacement from $n$ original observations, what is the approximate probability that any given original observation is not in the bootstrap sample (for large $n$)?

Solution

Each of $n$ draws is independent and excludes a given observation with probability $(n-1)/n$. The probability of being excluded from all $n$ draws is $(1 - 1/n)^n \to 1/e \approx \mathbf{0.368}$ for large $n$. That's why the out-of-bag sample of a random forest contains roughly $\mathbf{1 - 1/e \approx 63.2\%}$ of the training data.

Question 3 · explain

You pick the top 20 of 1000 features by correlation with $Y$, then run 10-fold CV on those 20 features. Your CV error looks great. Why is the number wrong?

Model answer

You leaked the label. Feature selection by correlation with $Y$ used the entire dataset, including what would later become validation folds. Each CV fold's "held-out" data was already used to pick the features that benefit the model. The fix: put feature selection inside the CV loop — for each fold, redo selection on that fold's training portion only. The correct estimate is almost always meaningfully worse.

Key pointspre-processing that uses $Y$ must live inside the CV loop; "CV on the pipeline, not the fitted features."

Question 4 · explain

Your coworker says "my bootstrap 95% CI for the correlation is [0.1, 0.9], so there's a 95% probability the true correlation is in that range." What's off?

Model answer

Same confusion as with any frequentist confidence interval. The bootstrap CI is a statement about the procedure: if you re-ran the whole study many times and constructed bootstrap CIs each time, 95% of them would cover the true correlation. This specific interval either contains the truth or it doesn't — the 95% is a property of the method, not a probability about this one number. Also, [0.1, 0.9] is so wide it carries almost no practical information — the honest message is "we can't pin this correlation down with this much data."

Key pointsCI is a method property, not a posterior; wide CI = low information.
