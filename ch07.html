Chapter 7

Moving Beyond Linearity

A line is rarely the truth. These tools let you stay inside the "linear regression" machinery while fitting curves, humps, and local wiggles.

Big idea

Replace each predictor with a flexible set of transformations, then run linear regression on those transformations.

Every method in this chapter is a clever way to build new basis functions — polynomials, step functions, splines — and feed them to the linear machinery. You get curves without leaving the framework you already know.

1. Polynomial regression

Add $X^2, X^3, \ldots, X^d$ as additional predictors. The fit is now a degree-d polynomial in X. Still linear in the parameters, still solved by least squares.

  • Pros: simple extension, captures smooth nonlinearity.
  • Cons: high-degree polynomials get wild at the edges of the data ("Runge's phenomenon"). Ugly things happen when extrapolating.
Past degree 4-5, polynomials are usually a bad idea. Splines (below) give you the flexibility without the tail explosions.

2. Step functions

Bin X into regions. In each bin, predict a constant. That's it — a staircase. Implementable by creating indicator variables for each bin and running linear regression.

Used when the relationship has natural breakpoints (age groups, income brackets) and you want clean "within this bin, the predicted value is X" interpretation. Wastes information within bins.

3. Splines — the right way to get a smooth curve

A spline is a piecewise polynomial glued together with smoothness constraints at the joints (called knots). You get local flexibility without global wildness.

Regression splines

Choose the number and location of knots. Fit cubic polynomials in each interval, requiring the function, its first derivative, and its second derivative to agree at each knot. Result: a smooth curve that adapts locally.

Natural cubic splines

Like cubic splines but with a constraint: linear beyond the outermost knots. This tames the tail behavior that ordinary polynomials suffer from. In practice, natural cubic splines are what you should reach for.

Smoothing splines

A different formulation. Instead of placing knots manually, put a knot at every data point and penalize wiggliness directly:

$$ \min_g \; \sum (y_i - g(x_i))^2 + \lambda \int g''(x)^2 \, dx $$

The integral of the squared second derivative measures how much the curve bends. $\lambda$ controls how smooth — high $\lambda$ gives a straight line, low $\lambda$ gives an interpolator.

5 knots

Regression spline with uniformly placed knots. More knots = more flexibility. Beyond a point you just chase noise.

4. Local regression (LOESS / LOWESS)

To predict at $x_0$, fit a low-degree polynomial using only the points nearby, weighted by proximity. Repeat at every point. Result: a smooth curve that adapts to local structure.

Knob: the span (fraction of points used). Small span = wiggly; large span = smooth. Basically a bias-variance knob.

Local regression is KNN regression with a polynomial fit at each neighborhood instead of a flat average. More graceful at the data edges.

5. Generalized Additive Models (GAMs)

A GAM lets each predictor have its own nonlinear function, then adds them up:

$$ Y = \beta_0 + f_1(X_1) + f_2(X_2) + \dots + f_p(X_p) + \epsilon $$

Each $f_j$ can be a smoothing spline, a polynomial, a step function — whatever. The key word is additive: no interactions by default (you'd add them manually).

Why GAMs are great

  • Flexible: each predictor can be nonlinear in its own way.
  • Interpretable: you can plot $f_j(X_j)$ for each predictor and read the story.
  • Linear in the functions: keeps a lot of the regression machinery (CIs, tests).

Why GAMs are not always great

  • No interactions by default — you have to opt in.
  • Don't scale as well as trees/NNs for very high-dimensional, low-signal data.
When GAMs shine: medium-dimensional problems where you suspect some variables bend but you want to see and defend every relationship. Medicine, economics, policy.

6. How to pick the flexibility

All these methods have a flexibility knob — polynomial degree, number of knots, smoothing parameter, span. All of them live and die by cross-validation (Ch 5) or an information criterion. Use it; don't eyeball.

Flashcards

1
Why is polynomial regression still "linear"?
Linear in the parameters β, not in X. You're doing linear regression on transformed predictors (X, X², X³, …).
2
What's wrong with high-degree polynomials?
They get wild at the edges of the data (Runge's phenomenon) and extrapolate badly. Use splines instead beyond degree 4-5.
3
What is a spline?
A piecewise polynomial where the pieces are joined at "knots" with smoothness constraints (continuous function and derivatives).
4
What does "natural" mean in natural cubic spline?
The spline is constrained to be linear beyond the outermost knots. This tames tail behavior and reduces variance at the edges.
5
What does λ control in a smoothing spline?
The tradeoff between fitting the data (RSS) and smoothness (integral of squared second derivative). High λ = straight line. Low λ = interpolator.
6
Describe LOESS in one sentence.
Fit a low-degree polynomial at each target point, using only the nearby points weighted by distance, to get a smooth curve.
7
What is a GAM?
A model of the form Y = β₀ + f₁(X₁) + f₂(X₂) + … where each fⱼ is a flexible smooth function. Additive: no interactions unless you add them.
8
GAM's main strength over a random forest?
Interpretability. You can plot each fⱼ(Xⱼ) and say "this is how variable j bends the prediction," which a forest can't tell you directly.

Quiz

Smooth flexibility without betraying interpretability.

Question 1 · compute

A cubic spline has one knot for every cubic piece boundary. For a cubic regression spline with $K$ knots, the number of parameters (including intercept) is $K + 4$. How many parameters for $K = 2$ knots?

Solution

$K + 4 = 2 + 4 = \mathbf{6}$. A natural cubic spline adds boundary constraints (linearity outside the data range) and gets down to $K + 2 = 4$ parameters for the same $K = 2$ — fewer effective degrees of freedom, more stable tails.

Question 2 · explain

Why use a natural cubic spline with 5 knots instead of a 10th-degree polynomial, if both have roughly the same effective flexibility?

Model answer

A 10th-degree polynomial is wildly unstable at the boundaries — its tails swing violently because a global polynomial has to bend far to accommodate faraway structure. A spline is piecewise cubic, so flexibility is local: a bump in one region doesn't warp the tails. A natural cubic spline further constrains the fit to be linear outside the boundary knots, which kills the tail instability entirely. Equal nominal flexibility, dramatically better extrapolation and stability.

Key pointspolynomial tails blow up; splines keep flexibility local; natural splines are linear outside boundary knots.

Question 3 · explain

A coworker fits a GAM and concludes "age has no effect" because its p-value is 0.3. But the partial plot of $f(\text{age})$ shows a clear U-shape. What's happening?

Model answer

The reported p-value is typically testing the null "the smooth function of age is identically zero" — or in some packages it's averaging over the whole smooth, which can average a positive half against a negative half toward zero. A U-shape has real effects at both ends but near-zero average. Trust the partial plot. If you want a formal test, test specific contrasts (e.g., $f(\text{age}=20)$ vs $f(\text{age}=50)$) or use a test for non-zero second derivative. A coefficient-style p-value can't summarize a non-monotonic smooth.

Key pointssingle p-value can't summarize a curve; U-shapes average to zero; trust the partial plot.
