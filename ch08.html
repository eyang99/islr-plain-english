Chapter 8

Trees, Forests, Boosting

A decision tree is a flowchart the model learns by itself. Alone it's weak. Add some clever ensembling and it becomes the most practically useful non-deep-learning tool in ML.

Big idea

One tree = a flowchart. Many trees averaged or boosted together = one of the best predictors you can buy.

A single tree is intuitive and interpretable but high variance. Bagging, random forests, and boosting are three different recipes for combining many trees into a single, lower-variance prediction machine.

1. The humble decision tree

A tree recursively splits the predictor space into rectangles. At each split, pick the predictor and threshold that best separates the data. In each leaf (terminal region), predict the mean of Y (regression) or the majority class (classification).

How a split is chosen

  • Regression: pick the split that minimizes RSS within the resulting two regions.
  • Classification: pick the split that minimizes a purity measure — usually the Gini index or entropy (cross-entropy). Both measure "how mixed is this node?"

Pros and cons of a single tree

  • Easy to explain ("if age > 40 and income > 50k, then …"). A boss can follow it.
  • Handles categorical variables and missing data naturally.
  • No need to scale features.
  • But: high variance. A different training sample gives a very different tree.
  • Piecewise-constant predictions — can't smoothly extrapolate.

2. Pruning

If you let a tree grow deep, it overfits (one leaf per observation, zero training error, terrible test error). The cure is cost-complexity pruning:

  1. Grow a large tree.
  2. For a range of penalty values $\alpha$, find the subtree minimizing $RSS + \alpha |T|$, where $|T|$ is the number of leaves.
  3. Pick $\alpha$ by cross-validation.

Same logic as ridge/lasso: trade some fit for less complexity.

3. Bagging — variance reduction by averaging

A single tree has high variance. If you had many independent datasets, you'd fit many trees and average them. But you don't. So use the bootstrap:

  1. Draw B bootstrap samples (sample n points with replacement).
  2. Fit a tree on each. Don't prune — let them overfit.
  3. Average predictions across trees (or take majority vote for classification).

Why it works: averaging reduces variance. If each tree is unbiased but high variance, the average is still unbiased but much lower variance.

Bagging loses the interpretability of a single tree. You can recover partial interpretability with variable importance — measure how much each variable reduces the splitting criterion across all trees.

4. Random forests — bagging with a twist

Problem with bagging: if one predictor is very strong, every tree's top split is on it, and the trees end up highly correlated. Averaging correlated things doesn't reduce variance much.

Random forest fix: at each split, only consider a random subset of predictors (typically $m = \sqrt{p}$ for classification, $p/3$ for regression). Weak predictors get more chances to split, trees are more diverse, averaging helps more.

Bagging makes B bootcamp instructors vote, but they all read the same playbook. Random forest blindfolds each instructor to different variables, forcing them to develop different approaches. The vote is more informative.

5. Boosting — learning slowly from mistakes

Different philosophy. Instead of training trees independently and averaging, train them sequentially, each one trying to fix what the previous ones got wrong.

Gradient boosting, in four lines

  1. Start with a constant prediction (the mean of Y).
  2. Compute the residuals (errors from the current prediction).
  3. Fit a small tree to predict those residuals.
  4. Add that tree's predictions (scaled by a small learning rate) to the current model. Repeat.

Three knobs:

  • B: number of trees. More = slower, more likely to overfit. Tune with CV.
  • λ (learning rate / shrinkage): how much each tree contributes. Small λ + large B generally wins.
  • d (interaction depth): tree depth. Usually small (1-8). d=1 is a "stump" and additive. Higher d allows interactions.
If bagging is a jury voting independently, boosting is a student taking the same test over and over, each time focusing harder on the questions they got wrong.

Why boosting often wins

By construction, each new tree targets the current weakness of the ensemble. Given enough time and right hyperparameters, it's hard to beat. XGBoost, LightGBM, CatBoost are all gradient boosting implementations optimized for speed and accuracy, and they dominate tabular-data leaderboards.

Boosting CAN overfit if B is too large. Unlike bagging, which just stops helping past a point, boosting keeps "learning" and can memorize noise. Always cross-validate B.

6. BART (Bayesian Additive Regression Trees)

Think of BART as boosting's Bayesian cousin. Each tree is tiny (forced to be weak by a prior), the ensemble is fit by a sampler, and you get uncertainty estimates for free. In practice it performs comparably to gradient boosting on many problems, and shines when you need predictive intervals.

Flashcards

1
How does a decision tree pick its splits?
Greedy: at each node, try every predictor and every threshold; pick the one that most reduces RSS (regression) or Gini/entropy (classification).
2
What does cost-complexity pruning do?
Grows a large tree then trims it, penalizing number of leaves: minimize RSS + α·|T|. Cross-validate α.
3
Why does a single tree have high variance?
Greedy splits are sensitive to small changes in the training data. A different sample of the same size often gives a very different tree.
4
What is bagging?
Bootstrap aggregating: fit B trees on B bootstrap samples, average (regression) or vote (classification). Reduces variance.
5
What's the twist that makes random forests better than plain bagging?
At each split, consider only a random subset of m predictors (typically √p). Decorrelates the trees so averaging reduces variance more.
6
Bagging vs boosting — core difference?
Bagging fits trees independently in parallel on bootstrap samples and averages. Boosting fits trees sequentially, each one correcting the previous ensemble's mistakes.
7
Three hyperparameters of gradient boosting?
Number of trees B, learning rate λ (shrinkage), tree depth d. Small λ + large B usually best; tune B via CV.
8
Can boosting overfit? Can bagging?
Boosting: yes, if B too large. Bagging: basically no — beyond a point, adding more trees just stops helping but doesn't hurt.
9
What's the main limitation of trees for extrapolation?
Predictions are constant within each leaf. Outside the training data range, you just get the nearest leaf average — no trend continuation.
10
Why use variable importance in a random forest?
You lose the single-tree story when you ensemble, but you can measure how much each variable contributes to splits across all trees — a proxy for "which features matter?"

Explain it back

Why does random forest typically outperform plain bagging on the same data?
Your boosting model's training error keeps dropping and test error is also dropping. Can you just keep adding trees until you're satisfied? What stops you?
You have a tabular dataset with 20 features and need to deploy a predictor. Would you rather ship a random forest or a gradient boosted model? What do you weigh?
