Chapter 4

Classification

The output is a category, not a number. Four ways to draw a boundary between classes, all doing roughly the same job with different assumptions.

Big idea

Classification is about estimating P(Y = class | X) and thresholding.

Every method in this chapter — logistic, LDA, QDA, naive Bayes — estimates the probability of each class given the inputs, then assigns you to the most probable class. They differ only in how they model those probabilities.

1. Why not just use linear regression?

You could code class labels as 0/1 and run linear regression. Two problems:

  1. Predictions leak outside [0, 1], so they can't be read as probabilities.
  2. With 3+ unordered classes (red/blue/green), coding them 1/2/3 imposes an arbitrary ordering.

So we need models that respect "this is a probability between 0 and 1."

2. Logistic regression

Instead of predicting Y directly, predict the probability of class 1 using the logistic (sigmoid) function:

$$ P(Y = 1 \mid X) = \frac{e^{\beta_0 + \beta_1 X}}{1 + e^{\beta_0 + \beta_1 X}} $$

The sigmoid squashes any real number into (0, 1). Negative inputs → close to 0. Positive → close to 1. Zero → 0.5.

β₁ controls steepness. β₀ shifts the curve left/right. Classification boundary = where the curve crosses 0.5.

Odds and the log-odds trick

Rearrange the sigmoid and take the log of both sides:

$$ \log\!\left(\frac{P(Y=1 \mid X)}{1 - P(Y=1 \mid X)}\right) = \beta_0 + \beta_1 X $$

The log-odds is linear in X. That's why we call it "logistic regression" — it's linear regression on the log-odds scale.

Reading the coefficients

$\beta_1 = 0.5$ means: a 1-unit increase in X multiplies the odds of class 1 by $e^{0.5} \approx 1.65$. Not the probability — the odds. People screw this up all the time.

Fitting: maximum likelihood

Unlike linear regression, there's no closed form. You maximize the likelihood:

$$ \ell(\beta) = \prod_{i: y_i=1} p(x_i) \prod_{i: y_i=0} (1 - p(x_i)) $$

Usually done by Newton-Raphson or gradient methods. Software does this for you.

3. Linear Discriminant Analysis (LDA)

Flip the problem around using Bayes' rule. Instead of modeling $P(Y \mid X)$ directly, model $P(X \mid Y)$ and the class prior $P(Y)$, then combine them.

$$ P(Y = k \mid X) = \frac{P(X \mid Y = k) \cdot P(Y = k)}{P(X)} $$

LDA assumes: within each class, X follows a multivariate Gaussian with the same covariance matrix across classes. Under that assumption, the boundary between classes turns out to be linear.

Imagine each class is a cloud of points shaped like a Gaussian. LDA assumes all clouds have the same shape and rotation — they just sit in different places. The decision line is exactly halfway between each pair of class centers (weighted by prior).

When LDA beats logistic regression

  • Classes are well-separated (logistic regression's estimates get unstable).
  • n is small and the Gaussian assumption is roughly OK.
  • You have more than 2 classes (LDA extends naturally; logistic needs multinomial extension).

4. Quadratic Discriminant Analysis (QDA)

Same as LDA, but drop the "same covariance across classes" assumption. Each class gets its own covariance matrix. Boundaries become quadratic (curved).

Use LDA when
  • Small n (QDA has many more parameters to estimate)
  • Classes plausibly share covariance
  • You want a simpler story
Use QDA when
  • Large n (can afford more parameters)
  • Classes clearly have different shapes
  • A linear boundary would miss structure

It's a bias-variance tradeoff again. LDA has higher bias (the equal-covariance assumption) but lower variance. QDA is more flexible but needs more data.

5. Naive Bayes

Same Bayes-rule setup as LDA, but with a drastic simplification: assume that, within each class, the features are independent of each other.

$$ P(X_1, X_2, \dots, X_p \mid Y = k) = \prod_{j=1}^p P(X_j \mid Y = k) $$

This is almost never true. Age and income aren't independent inside "bought product" either. But the model still works remarkably well, because what you lose in accuracy of the probabilities, you gain in stability from fewer parameters.

Naive Bayes is the Mr. Magoo of classifiers — its assumptions are absurd, but somehow it bumbles into the right answer often enough to be useful.

Why it's especially useful

  • High-dimensional data (text classification, spam filtering).
  • Very small training sets where richer models overfit.
  • Mixed categorical and continuous features (easy to plug in per-feature densities).

7. Confusion matrix, sensitivity, specificity

Accuracy = (correct predictions) / (total). But it can hide a lot. If 99% of emails are not spam, a model that always says "not spam" gets 99% accuracy and catches zero spam. Useless.

The confusion matrix

Predicted +Predicted −
Actual +TP (true positive)FN (false negative)
Actual −FP (false positive)TN (true negative)
  • Sensitivity (recall, TPR) = TP / (TP + FN). Of actual positives, how many did I catch?
  • Specificity (TNR) = TN / (TN + FP). Of actual negatives, how many did I correctly dismiss?
  • Precision = TP / (TP + FP). Of my positive predictions, how many are right?

You can trade sensitivity for specificity by changing the threshold. The ROC curve plots sensitivity vs (1 − specificity) at every threshold. AUC (area under the curve) summarizes performance across all thresholds — 0.5 is random, 1.0 is perfect.

Sensitivity = "don't miss the sick patients." Specificity = "don't scare healthy patients." A test for a rare deadly disease should have high sensitivity even at the cost of specificity. A test for a common cold can prioritize specificity. Pick the knob based on what the mistake costs.

Flashcards

1
Why can't you use plain linear regression for classification?
Predictions escape [0,1] (not valid probabilities) and with 3+ categories, numerical codes impose a false ordering.
2
What does the sigmoid function do?
Squashes any real number to (0, 1) so the output is a valid probability. It's S-shaped.
3
In logistic regression, what does β₁ mean?
A 1-unit increase in X multiplies the ODDS of class 1 by e^β₁. Not the probability. Odds.
4
State LDA's key assumption.
Within each class, X is multivariate Gaussian, and all classes share the same covariance matrix. That assumption makes the decision boundary linear.
5
LDA vs QDA, trade-off?
LDA: fewer parameters, linear boundary, higher bias, lower variance. QDA: per-class covariance, quadratic boundary, lower bias, higher variance. LDA for small n, QDA for large n.
6
What's "naive" about Naive Bayes?
Assumes features are independent within each class. Usually false, but the simplification makes the model stable and fast, and it still works well on many problems.
7
Define sensitivity and specificity.
Sensitivity (TPR) = TP / (TP + FN) — fraction of actual positives caught. Specificity (TNR) = TN / (TN + FP) — fraction of actual negatives correctly dismissed.
8
What does the ROC curve show?
Sensitivity vs (1 − specificity) across every classification threshold. AUC summarizes it: 0.5 is random, 1.0 perfect.
9
Why is accuracy a bad metric for imbalanced classes?
If one class is 99%, always predicting it gets 99% accuracy while being useless. Use sensitivity/specificity/F1 or AUC instead.
10
What's the Bayes decision rule?
Assign x to the class with highest P(Y=k | x). This minimizes error. LDA/QDA/Naive Bayes are all approximations of this rule.
